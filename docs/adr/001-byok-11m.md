# ADR-001: Bring Your Own Key (BYOK) LLM Design

## Status
Accepted

## Date
2026-04-10

## Context
PhishHawk Requires an LLM for IOC correlation, actor profiling, and report generation. Hosting an LLM er managing API keys centrally would require PhishHawk to operate as a service - incurring costs, infrastructure overhead, and responsibility for user data. This conflicts with the project's goal of being a simple, self-hosted open source tool.

## Decision
PhishHawn will not provide or manage LLM access. Instead, users supply their own API key via `config.yaml`. The tool supports multiple LLM providers through a common adapter interface:

- **Anthropic** (Claude)
- **OpenAI** (GPT-4)
- **Ollama** (local models, no API key required)

The active provider and model are configured in `config.yaml`:

```yaml
llm:
provider: anthropic
api_key: YOUR_API_KEY
model: claude-sonnet-4-20250514
```

## Consequenses

**Positive**
- Zero LLM hosting costs for the project
- Users retain full controll over their API keys and data
- Local Ollama support enables fully air-gapped deployments
- Multi-provider support reduces vendor lock-in

**Negative**
- Users must obtain and manage their own API keys
- LLM outout quality vaires between providers
- Ollama requires local GPU/ CPU resources

## Alternatives Considered
- **Hosted API with shared key** - rejected due to cost and data privacy concerns
- **Single provider only** - rejected to avoid vendor lock-in and support air-gapped environments
